## 🏨 Описание на проекта
 
**Hotel_Manager** представлява уеб базирана система за управление на хотелски резервации, стаи и допълнителни услуги. Основната цел на проекта е да се създаде централизирана платформа, чрез която администратори и рецепционисти могат да управляват хотелските операции, а потребителите да проследяват своите резервации.
 
Чрез използването на **MVC архитектура** и **структурирана релационна база данни**, системата осигурява:
- ефективна работа
- лесна поддръжка
- възможност за бъдещо разширяване
 
---
 
## 🧱 Структура на проекта
 
Системата е изградена по **MVC архитектурен модел**, който разделя приложението на:
- слой за представяне (Views)
- слой за бизнес логика (Services)
- слой за достъп до данни (Data)
 
### 📊 Съхранение на данни
 
Системата съхранява информация за:
- Типове стаи
- Индивидуални стаи
- Резервации
- Допълнителни услуги
- Потребители и роли
- Автомобили на потребителите
 
Използвана е релационна база данни, реализирана чрез **Entity Framework Core**, като управлението на схемата се осъществява чрез миграции.

## 🗄 Описание на таблиците
 
### 1. `ApplicationUser`
- Id – Primary Key
- FirstName
- LastName
- Age
- IsActive
- CreatedAt
 
### 2. `UserCar`
- Id – Primary Key
- LicensePlate
- UserId – FK към ApplicationUser
 
### 3. `RoomType`
- Id – Primary Key
- Name
- Capacity
- PricePerNight
 
### 4. `Room`
- Id – Primary Key
- RoomNumber
- IsAvailable
- RoomTypeId – FK към RoomType
 
### 5. `Reservation`
- Id – Primary Key
- CheckInDate
- CheckOutDate
- TotalPrice
- Status
- CreatedAt
- UserId – FK към ApplicationUser
 
### 6. `HotelService`
- Id – Primary Key
- Name
- Price
 
### 7. `ReservationRoom` (Many-to-Many)
- ReservationId – FK
- RoomId – FK
 
### 8. `ReservationService` (Many-to-Many)
- ReservationId – FK
- ServiceId – FK
